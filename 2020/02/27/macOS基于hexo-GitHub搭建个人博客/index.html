macOS基于hexo+GitHub搭建个人博客

Posted on 2020-02-27 Edited on 2020-02-29 Valine:

Hexo官网

1 环境配置

1.1 安装node.js

node.js官网

Javascript运行环境，用于生成静态页面

至少Node.js 8.10，推荐10.0或更高

1.2 git

Mac系统自带

2 安装并配置Hexo

2.1 用npm命令安装hexo

$ npm install -g hexo //如提示无权限，指令前加sudo

检查安装成功

![截屏2020-02-27下午1.54.11](/Users/jean/Library/Application Support/typora-user-images/截屏2020-02-27下午1.54.11.png)

2.2 本地初始化

在本地创建一个文件夹(eg：Blog)，cd到该路径下，执行下列语句，向文件夹内复制建立网站所需的文件

$ hexo init

待以上初始化执行完，安装依赖

$ npm install

编译站点

$ hexo generate

启动本地hexo server服务

$ hexo server

![截屏2020-02-27下午2.08.38](/Users/jean/Library/Application Support/typora-user-images/截屏2020-02-27下午2.08.38.png)

根据提示打开http://localhost:4000/

![截屏2020-02-27下午2.09.43](/Users/jean/Library/Application Support/typora-user-images/截屏2020-02-27下午2.09.43.png)

tada！

3 链接GitHub

3.1 创建新仓库

命名为“username.github.io” //遵循GitHub Pages的命名原则

![截屏2020-02-27下午2.14.18](/Users/jean/Library/Application Support/typora-user-images/截屏2020-02-27下午2.14.18.png)

3.2 修改部署文件

找到本地博客文件夹中的_config.yml文件并修改

![截屏2020-02-27下午6.31.23](/Users/jean/Library/Application Support/typora-user-images/截屏2020-02-27下午6.31.23.png)

type：使用什么方式发布站点

repo：发布的目的地。注意我这里是配置了SSH-key后的形式，未配置的请参考http形式。

branch：使用哪个分支

3.3 安装插件

为了将hexo部署至GitHub上，我们需要一个插件

$ npm install hexo-deployer-git —-save

然后重新运行

至此，通过username.github.io即可访问你的主页了

4 页面美化

GitHub上提供了大量hexo主题，这里采用NexT
详情参阅//此处应有一个页面布置的链接

5 发布博客

5.1 创建博客

还是在本地博客文件夹目录下

$ hexo new post '文章标题'

打开文件夹->source->_posts，可以看到我们刚刚创建的blog，格式为markdown

//安利一个markdown编辑器Typora，支持macOS，Windows，Linux

将文章编辑完成后，重复上面三条指令，即可更新至GitHub。
